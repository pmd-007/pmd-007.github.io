🔍Path Traversal

1. Path là gì?

Trong đời thực, path giống như đường đi từ nhà bạn ra chợ: có thể là đường tuyệt đối (absolute) hoặc đường tương đối (relative). Trên máy tính cũng vậy: path là đường dẫn đến file hoặc thư mục.

Ví dụ:

Absolute Path Relative Path
/etc/ssh ./ssh hoặc ssh

Trong Linux có vài quy ước nhỏ nhưng quan trọng:

  • . → thư mục hiện tại
  • .. → thư mục cha

👉 Nghe thì đơn giản, nhưng chỉ cần hacker biết cách lợi dụng mấy ký hiệu này, bạn sẽ thấy “đi lạc” thành một cuộc tấn công thật sự.

2. Path Traversal là gì?

Path Traversal (hay Directory Traversal) là kỹ thuật hacker chèn thêm ../ vào đường dẫn để thoát khỏi thư mục được phép và truy cập sang chỗ nhạy cảm.

Ví dụ đoạn code PHP:

$file_name = $_GET['file_name'];
$file_path = '/var/www/html/images/' . $file_name;

if (file_exists($file_path))
    readfile($file_path);
else
    echo '404 Not Found';

Ở đây $_GET['file_name'] chính là Untrusted Data. Nếu hacker nhập ../../../etc/passwd, biến $file_path sẽ thành:

/var/www/html/images/../../../etc/passwd

👉 Và file /etc/passwd lộ ra ngoài 😱

3. Tại sao nguy hiểm?

  • Hacker có thể đọc file nhạy cảm: /etc/passwd, config DB, source code.
  • Nếu có hàm ghi file → có thể ghi đè file hệ thống → dẫn đến RCE.
  • Người dùng vô tội có thể bị ảnh hưởng: lộ thông tin, phishing, chiếm quyền.

Tóm gọn: chỉ cần một input không kiểm soát, hậu quả có thể lan rộng cả hệ thống.

4. Biến thể của Path Traversal

Một số chỗ dễ dính Path Traversal trong code PHP:

  • READ → readfile(untrusted_data)
  • WRITE → file_put_contents(untrusted_data)
  • DELETE → unlink(untrusted_data)
  • RCE → include(untrusted_data)

👉 Chỉ cần một chỗ sai, hacker có thể biến một input nhỏ thành một cuộc tấn công nghiêm trọng.

5. Path Traversal trong file nén (Zip Slip)

Path Traversal không chỉ xảy ra trong web, mà còn trong việc giải nén file ZIP/TAR.

Ví dụ code Java:

File f = new File(destinationDir, e.getName());
IOUtils.copy(input, write(f));
  • e.getName() → tên file trong zip (Untrusted Data).
  • Nếu hacker nhét file có tên ../../../tmp/test, thì khi giải nén, file sẽ nằm ở /tmp/test.

👉 Đây là kỹ thuật Zip Slip, từng làm mưa làm gió trong cộng đồng bảo mật.

6. Một ví dụ thực tế – CDNJS Cloudflare Zip Slip

Bug hunter phát hiện lỗi Path Traversal trong thư viện archive/tar (Golang). Vấn đề: server không kiểm tra tên file trong .tar.gz khi giải nén.

Hậu quả:

  • Hacker upload .tar.gz độc hại.
  • Khi giải nén, file được ghi đè vào vị trí nhạy cảm.
  • Nếu ghi đè file script chạy định kỳ → dẫn đến RCE.

Chi tiết: Cloudflare Zip Slip to RCE

7. Cách phòng thủ ✍️

  • Luôn giả định input liên quan đến path/file là không tin cậy.
  • Nghĩ ngay: “Nếu mình thêm ../ vào đây thì sao?”
  • Phòng thủ:
    • Normalize path trước khi dùng.
    • Áp dụng whitelist cho file hợp lệ.
    • Không bao giờ trust input trực tiếp.
