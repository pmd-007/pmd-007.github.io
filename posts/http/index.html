🌐 HTTP – Giao thức nền tảng của Web

Nếu web là một thành phố khổng lồ, thì HTTP chính là những con đường để xe (dữ liệu) chạy từ nhà bạn (browser) tới cửa hàng (server). Nhưng những con đường đầu tiên… không có đèn giao thông, cũng chẳng có bảo vệ. Hãy cùng tìm hiểu “ông tổ” của Internet trước khi lên đời HTTPS nhé!

🏗️ HTTP là gì?

  • HTTP (HyperText Transfer Protocol): Giao thức lớp ứng dụng (Application Layer) dùng để trao đổi dữ liệu giữa client và server.
  • Ra đời năm 1991, HTTP ban đầu chỉ tải văn bản, giờ thì kéo cả video, livestream và game.
  • Mô hình Client–Server:
    • Client (browser, app) gửi request.
    • Server xử lý và trả về response.
  • Port mặc định: 80.

💡 Nghĩ đơn giản: Bạn gọi món ở quán phở (HTTP Request), bếp nấu và bưng phở ra (HTTP Response).

🔍 Cách HTTP hoạt động

Client (Browser) ---> HTTP Request ---> Server
Client <--- HTTP Response <--- Server

Ví dụ truy cập http://example.com:

Request:

GET /index.html HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0
Accept: text/html

Response:

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html>...</html>

📌 Tất cả đều là plain text → Hacker “ngồi bàn bên” đọc được menu, biết bạn gọi phở gà hay phở bò. 🥲

🧾 Các phương thức HTTP – “Từ gọi món tới đập bàn”

Method Ý nghĩa Ví dụ đời thường
GET Lấy dữ liệu Xem menu quán ăn.
POST Gửi dữ liệu Đặt món + ghi chú “ít hành”.
PUT Cập nhật tài nguyên Đổi món phở tái thành phở chín.
DELETE Xóa tài nguyên Gạch món khỏi order.
HEAD Lấy thông tin, không lấy nội dung Hỏi nhân viên: “Quán còn phở bò không?” mà không gọi món ngay.
OPTIONS Hỏi server “chị bán món gì?” Kiểm tra quán có món nào trước khi đặt.
PATCH Sửa một phần tài nguyên Thêm trứng chần vào tô phở đang ăn.

📡 HTTP Status Codes – “Bảng đèn báo”

Nhóm mã Ý nghĩa Ví dụ
1xx Thông tin 100 Continue
2xx Thành công 200 OK
3xx Chuyển hướng 301 Moved
4xx Lỗi từ phía client (bạn gõ sai) 404 Not Found
5xx Lỗi từ phía server (quán cháy bếp) 500 Internal

💡 Gặp 404 Not Found = “Xin lỗi anh, hết phở bò rồi!”

🪪 Đặc tính của HTTP

  • Stateless: Mỗi request như một khách mới → quán không nhớ bạn vừa gọi gì nếu không dùng cookie/session.
  • Đơn giản: Dễ debug, dễ học.
  • Không bảo mật: Tất cả dữ liệu gửi đi không mã hóa → hacker có thể “nghe lén”.

📦 Các phiên bản HTTP

Phiên bản Năm ra mắt Đặc điểm
HTTP/0.9 1991 Chỉ có GET, trả về text thuần.
HTTP/1.0 1996 Thêm header, hỗ trợ nhiều content.
HTTP/1.1 1997 Keep-Alive, caching, vẫn phổ biến.
HTTP/2 2015 Multiplexing, nén header, nhanh hơn.
HTTP/3 2022 Dựa trên QUIC/UDP, độ trễ thấp, siêu mượt.

📌 Nếu HTTP/1.0 là xe máy đời cũ, thì HTTP/3 là “siêu xe đường đua”.

🚨 Nguy cơ bảo mật

  • Nghe lén (Sniffing): Hacker có thể đọc tất cả thông tin bạn gửi.
  • MITM (Man-in-the-Middle): Kẻ xấu chặn đường, bỏ thuốc độc vào tô phở (response).
  • Session Hijacking: Cookie bị trộm → hacker đăng nhập tài khoản của bạn.

💡 HTTP trần trụi như đi chợ không mang ví khóa kéo – tiện nhưng nguy hiểm.

🔧 Công cụ học và test HTTP

  • cURL: “Đồ nghề cơ bản” để gửi request nhanh: 
    curl -v http://example.com
  • Burp Suite / Postman: Debug API, kiểm tra bảo mật web.
  • Wireshark: Bắt gói tin, nhìn rõ dữ liệu HTTP như… tin nhắn phơi ngoài chợ.

📝 Ghi nhớ

  • HTTP là nền móng của web, nhưng trần trụi, dễ bị nghe lén.
  • Dữ liệu (user/pass, cookie) đều plain text → dễ bị khai thác.
  • Đây là lý do ra đời HTTPS – phần tiếp theo sẽ “mặc giáp” cho HTTP.
