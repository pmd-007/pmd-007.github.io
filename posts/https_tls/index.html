🔒 TỪ “HTTP TRẦN” ĐẾN “HTTPS MẶC GIÁP”: LỊCH SỬ, CƠ CHẾ, VÀ ĐỜI THƯỜNG

Bạn từng lướt web và thấy trên thanh địa chỉ có trang bắt đầu bằng http:// (chả có gì) và có trang https:// với chiếc ổ khóa 🔒 chưa? Đằng sau cái “s” bé xíu đó là cả một lịch sử an ninh mạng dài cả thập kỷ. Hãy cùng đi từ thời “trẻ trâu Internet” tới “siêu chiến binh TLS 1.3” nhé!

🕰️ LỊCH SỬ HTTPS/SSL/TLS

1994 ──── SSL 1.0: Netscape làm thử, bảo mật yếu nên giấu luôn.
1995 ──── SSL 2.0: Lộ hàng, hacker MITM thoải mái nghe lén.
1996 ──── SSL 3.0: Bước tiến lớn, nhưng 2014 bị hạ bởi POODLE 🐩.
1999 ──── TLS 1.0: Chuẩn hóa SSL, mở ra kỷ nguyên TLS.
2006 ──── TLS 1.1: Vá vài lỗi, nhưng vẫn không đủ mạnh.
2008 ──── TLS 1.2: “Best seller” suốt 10 năm, AES-GCM, SHA-256.
2018 ──── TLS 1.3: Tốc độ bàn thờ, bảo mật top, tối giản handshake.

📌 Nhớ: SSL = tiền thân của TLS. Từ 2018 trở đi, ai không dùng TLS 1.2+ là “sống trong quá khứ”.

🤝 HANDSHAKE TLS

Hãy tưởng tượng:

  • Bạn (Client) muốn nói chuyện bí mật với một người bạn (Server).
  • Hai bạn không ai tin ai, xung quanh đầy kẻ soi mói (Hacker).

Quy trình handshake như sau:

Client (Browser)                        Server (Website)
-----------------                       ----------------
      |                                          |
      | 1. ClientHello: "Mình dùng TLS 1.3, thích AES-GCM" |
      |----------------------------------------->|
      |                                          |
      | 2. ServerHello: "Ok, mình cũng TLS 1.3"   |
      |<-----------------------------------------|
      |                                          |
      | 3. Certificate: "Đây là CMND số, CA ký rồi!" |
      |<-----------------------------------------|
      |                                          |
      | 4. ECDHE Key: "Đây là public key của mình" |
      |<-----------------------------------------|
      |                                          |
      | 5. Client gửi Pre-Master Key (mã hóa)    |
      |----------------------------------------->|
      |                                          |
      | 6. Cả hai tạo Session Key chung          |
      |                                          |
      | 7. Finished: "Ổn rồi, bật kênh bảo mật!" |
      |<---------------------------------------->|
      |                                          |
      | 🔐 Trò chuyện an toàn!                   |

📌 TLS 1.3 chỉ cần 1 vòng (1-RTT) bắt tay. TLS 1.2 cần 2 vòng (2-RTT) → chậm hơn.

🧩 GIẢI MÃ THUẬT TOÁN

Thuật toán Loại Vai trò Ví dụ đời sống
RSA Bất đối xứng Trao đổi khóa (cũ) Gửi chìa khóa trong két sắt – chỉ người có chìa riêng mới mở được.
ECDHE Bất đối xứng Sinh khóa phiên có PFS Hai người hẹn tạo mật mã bí mật ngay giữa quán cà phê mà không ai nghe được.
AES-GCM Đối xứng Mã hóa dữ liệu nhanh, xác thực Chat bí mật siêu tốc độ, lại có chữ ký chống giả mạo.
ChaCha20-Poly1305 Đối xứng Tối ưu mobile Nhắn tin bí mật mà điện thoại “cùi bắp” vẫn chạy mượt.
SHA-256/HMAC Hash/MAC Kiểm tra toàn vẹn Tem chống giả trên gói hàng – bóc ra là biết ngay.

🧪 DEMO CODE: TẠO KẾT NỐI HTTPS VÀ XEM CHỨNG CHỈ

Ví dụ Python: kết nối tới https://example.com, xem chứng chỉ TLS.

import ssl       # Thư viện chuẩn Python, hỗ trợ SSL/TLS
import socket    # Dùng để tạo kết nối mạng mức thấp (TCP socket)
import pprint    # In dữ liệu ra đẹp, dễ đọc

# Tên host cần kết nối
hostname = 'example.com'

# Tạo context TLS mặc định (chứa bộ CA tin cậy, cấu hình bảo mật an toàn)
context = ssl.create_default_context()

# Mở kết nối TCP tới server, port 443 (HTTPS)
with socket.create_connection((hostname, 443)) as sock:
    # Bọc kết nối TCP trong lớp SSL/TLS (kích hoạt handshake)
    with context.wrap_socket(sock, server_hostname=hostname) as ssock:
        # Lấy thông tin chứng chỉ số (X.509)
        cert = ssock.getpeercert()
      
        # In ra phiên bản TLS đang dùng
        print(f"Connected to {hostname} with {ssock.version()}")
      
        # In chi tiết chứng chỉ
        pprint.pprint(cert)

💡 Code trên:

  • Tạo kết nối TCP tới example.com:443.
  • Khởi động SSL/TLS, in ra thông tin chứng chỉ và phiên bản TLS.

🌍 VÍ DỤ ĐỜI THƯỜNG

  1. Thanh toán online:

    • HTTP: Bạn đọc to số thẻ tín dụng giữa chợ.
    • HTTPS: Bạn viết số thẻ, bỏ vào két, khóa lại, đưa cho người bán.

  2. Đăng nhập Wi-Fi công cộng:

    • HTTP: Hacker trong quán cà phê có thể đọc user/pass như đọc báo.
    • HTTPS: Hacker chỉ thấy ký tự loằng ngoằng không giải nổi.

  3. Website với ổ khóa vs không ổ khóa:

    • 🔒 Có HTTPS: Bạn đang nói chuyện với đúng người cần nói.
    • ❌ Chỉ HTTP: Có thể bạn đang nói chuyện với “anh hàng xóm giả dạng”.

🔑 TÓM TẮT LỊCH SỬ

Thời kỳ Thuật toán chính Nhận xét
1990s (SSL) RC4, DES, 3DES, MD5 Yếu như cửa gỗ mục.
2000s (TLS 1.0/1.1) AES (CBC), SHA-1 Còn dùng nhưng không khuyên.
2010s (TLS 1.2) AES-GCM, ECDHE, SHA-256 Tiêu chuẩn an toàn.
2020s (TLS 1.3) AES-GCM, ChaCha20 Nhanh, bảo mật, “best practice”.

🏆 KẾT LUẬN

  • HTTPS = HTTP + “áo giáp” TLS.
  • TLS 1.3 là chuẩn hiện nay: nhanh, bảo mật, thân thiện.
  • Đừng coi thường chữ “S” – nó cứu bạn khỏi hacker!
